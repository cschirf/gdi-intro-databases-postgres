Intro to SQL

  • Stands for Structured Query Language
  • Special purpose language for managing data in a RDBMS
  • The basics are the same, but each RDBMS will have its own dialect
  • Based on relational algebra
  • First introduced by Oracle in 1979
  • SQL is an ANSI and ISO standard
  • SQL is great to know, because it is everywhere
  • Fun fact! Cassandra uses a syntax eerily similar to SQL

Why PostgreSQL?

  • Open source & free
  • Widely used in industry
  • Well behaved RDBMS (does what you would expect)
  • Has full set of functionality
  • Handles concurrency and high loads well
  • Well documented
  • Supported by major languages and ORMs (object relational mappers)

Other popular RDBs

  • MySQL acquired by Oracle, but open source & free
  • Sqlite open source & free; good for mobile applications
  • SQL Server proprietary db owned by Microsoft
  • Oracle proprietary db owned by Oracle

Database Schemas

  • A structure of the database, described in a formal language
  • Acts as a blueprint for how the database is constructed
  • In a relational database, the schema defines:
    • tables
    • fields
    • relationships
    • views
    • indexes
    • functions
    • triggers
    • stored procedures
    • and other elements

SELECT

  • We use the SELECT command to retrieve data from a database
  • The output from a SELECT statement is always a grid of rows and columns
  • You do not have to capitalize SQL keywords like SELECT. This is a style preference.
  • Some of the later examples will show parts of the query on different lines. This is purely for readability purposes. SQL is not sensitive to whitespace.

SELECT

  • The most simple SELECT statement retrieves all values from a single table
  • For this statement, we use a * (asterisk, read “star”) to tell SQL to return every column in the table
  • We also won’t specify any filter criteria, so SQL will also return every row or record in the table
    • furbabies# SELECT * FROM person;

Specify columns

  • Instead of using *, we can tell SQL which columns we want to see by specifying the names after SELECT
  • Columns must be separated by commas (,)
  • Things to know: names of database entities (like columns, tables, databases) cannot contain spaces

common WHERE Clause Operators

Operator Description
= Equal
IS Another way to specify equals, but usually only works for booleans and nulls
<> Not equal
!= Not equal
> Greater than
< Less than
>= Greater than or equal
<= Less than or equal
BETWEEN Between an inclusive range
LIKE Search for a pattern. The wildcard character is %. Things to know: By default, postgres is case-sensitive. If you want to check for a pattern, but don’t care about case, use the ILIKE operator.
IN (value, value, ...) To specify multiple possible values for a column
NOT Negate the condition that comes after
AND Combine criteria
OR Another way to combine criteria
